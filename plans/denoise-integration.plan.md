# Denoise Integration: Expose stack context for denoise integration

## Overview

Implement a stable, machine-readable contract for stack artifacts generated by
`dn init stack` to enable denoise and other external tools to discover and use
milestone stack data without brittle markdown parsing.

## Issue Context

- Issue: #1
- Description: Expose stack context for denoise integration. Denoise needs to
  detect `dn init stack` artifacts in repositories and surface actionable UI.
  Requires stable contract for filename pattern, frontmatter sections, and score
  semantics.
- Labels: (none)

## Implementation Plan

### Phase 1: Enhance Frontmatter Contract

1. Add new frontmatter fields to `formatPlanFile()` in `cli/init-stack.ts`:
   - `milestone_title`: the milestone's title (already in body, add to
     frontmatter)
   - `issue_count`: number of scored issues
   - `generated_at`: ISO timestamp (complement `updated` date field)

2. Create companion JSON index file alongside the stack markdown:
   - Path: `plans/{milestone-number}.stack.json`
   - Contains: full machine-readable data for each issue (score, title, ref)

### Phase 2: Add JSON Index Generation

1. Create `writeStackIndex()` function in `cli/init-stack.ts`:
   - Output `plans/{milestone-number}.stack.json`
   - Schema:
     ```json
     {
       "version": "1.0",
       "milestone": { "number": 42, "title": "Q2 Features" },
       "repo": "owner/repo",
       "generated_at": "2026-04-23T12:00:00Z",
       "issues": [
         {
           "ref": "#45",
           "title": "Add login button",
           "score": 1,
           "disqualified": false
         },
         {
           "ref": "#50",
           "title": "Complex refactor",
           "score": null,
           "disqualified": true,
           "reason": "..."
         }
       ],
       "kickstart_order": ["#45", "#42", "#41"]
     }
     ```

2. Write JSON index on every `dn init stack` execution (both create and refresh)

### Phase 3: Document Scoring Contract

1. Document scoring extraction semantics in `kickstart/system.prompt.score.md`
   or a new integration doc:
   - Fibonacci scale: 1 (easiest/one-shot) through 8 (hardest/many iterations)
   - Scores are derived from LLM analysis of issue body and comments
   - Missing score = issue disqualified or not scored
   - Tie-breaking: sort by score ascending (easiest first)

2. Add scoring semantics to frontmatter/JSON for programmatic access

### Phase 4: Add Integration Documentation

1. Create `docs/denoise-integration.md` with integration guidance:
   - File discovery patterns
   - Frontmatter contract
   - JSON index schema
   - Code examples for reading stack data

2. Update `docs/subcommands.md` to point to new integration doc

### Phase 5: Add Stack Detection Utilities (Optional)

Consider adding a simple detector function in `sdk/` for external tools:

- `sdk/stack.ts`: `detectStackArtifacts(dir: string): StackArtifact[]`
- Returns discovered stack files with parsed metadata
- Not required for v1 but enables clean integration
